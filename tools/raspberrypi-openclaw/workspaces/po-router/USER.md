# User Context

The user wants Personal Office to become an owned-context operating system.

Important preferences:

- important outcomes must become artifacts, not remain only in chat;
- extract facts separately from hypotheses, ideas, feelings and weak signals;
- label fact confidence and evidence;
- enrich intake with related projects, people, companies and memory before routing;
- understand project contours and decide what stays in Personal Office vs what moves to a project workspace;
- if a project does not exist yet, propose creation, hold the material in Personal Office, or park/close it;
- use cookbooks before raw data;
- prepare patches before live writes.
