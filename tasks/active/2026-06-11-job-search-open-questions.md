# Job Search Open Questions

- Created: 2026-06-11
- Status: active
- Area: personal-projects / personal-brand / job-search process
- Related workspace: `personal-projects/personal-brand/workspace/`

## Context

The search framework is now mapped, but several decision rules are still unresolved. These questions need explicit answers before we can trust the search system to make good choices consistently.

## Open Questions

### Search Policy

- When do we not apply at all?
- When do we use the standard HH CV vs a tailored copy?
- When do we stop after the first recruiter signal and wait?
- How strict should the apply / maybe / skip boundary be?

### Seniority / Profile Signal

- When should we keep the full `23 years` story?
- When should we downshift to `12-15 years`?
- How do we write hands-on leadership without sounding downgraded?
- What phrasing is too “executive” for HH or recruiter screens?

### Market Logic

- Which roles are a real fit for the Russian market vs international market?
- How should we classify integrator roles?
- How should we classify business-unit roles?
- Where is the tech-stack gap acceptable and where is it a hard stop?

### Economics

- What is the minimum acceptable salary?
- What is the target salary by market?
- What relocation package threshold makes a move rational?
- When does self-relocation still make sense?

### Relocation

- Which countries are only okay with strong package support?
- Which countries are boundary cases rather than true targets?
- Which cities or pockets deserve an exception even if the country average is weaker?

## Desired Outcome

Produce a short decision matrix and fallback rules so future vacancy analysis can resolve these questions quickly.

## Next Step

Work through the open questions one by one and turn them into a reusable rules note.
