# Task Rules

## Task Template

```md
# Task Title

Status: Active
Created: YYYY-MM-DD
Area:
Owner:
Due:

## Context

## Desired Outcome

## Next Step

## Waiting On

## Links
```

## Statuses

- `Active` - doing or next action exists.
- `Waiting` - blocked by another person/event.
- `Someday` - useful but not committed.
- `Done` - completed with outcome captured.

## Rules

- Every active task needs a concrete next step.
- Every waiting task needs who/what it is waiting on.
- Done tasks should capture the result, not just say "done".
