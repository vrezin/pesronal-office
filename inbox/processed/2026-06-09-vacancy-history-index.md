# 2026-06-09 - Vacancy History Index

## Decision

Reviewed vacancies need an operating index, not only separate JD archives and analyses.

## Artifact

- `personal-projects/personal-brand/workspace/job-intake/INDEX.md`
- `personal-projects/personal-brand/workspace/job-intake/COMPANY_NOTES.md`

## Rule

Every new job-intake analysis should update the index with:

- date;
- company;
- role;
- primary track;
- recommended CV;
- decision;
- priority;
- next action;
- link to analysis.

If the intake contains a named company or changes our view of an unnamed company bucket, update `COMPANY_NOTES.md` too.

## Reason

The index prevents repeated analysis, preserves prior rejection/filter patterns, and gives agents a fast memory surface before selecting a CV or drafting a response. Company notes preserve what we have inferred about employers, channels, and recurring unnamed buckets.
