# Preferences

## Repository Work

- Important outcomes should become repository artifacts, not remain only in chat.
- Prefer repo-relative aliases over machine-specific absolute paths.
- Keep personal, company, and public-brand contours separated.
- Use practical markdown/jsonl first; add heavier infrastructure only when the file-backed layer becomes insufficient.

## Planning

- Plans should be built from current facts, constraints, obligations, and capacity.
- Do not hardcode yesterday's plan into reusable planning rules.
