# Vacancy Review Index

Updated: 2026-07-04

This is the operating index of reviewed vacancies. It summarizes the detailed artifacts in `analyses/` and links each row back to the full analysis.

## How To Use

- Before analyzing a new vacancy, check this index for similar roles, repeated companies, prior rejections, CV choices, and risk patterns.
- After creating a new analysis, add or update one row here.
- For new or updated rows, make the effort class visible in `Priority` or `Next`: `A-class` for rare strong fits, `B-class` for fast template applications, `C-class` for parked/maybe roles.
- The detailed analysis remains the source of truth for evidence and cover-letter text.

## Current Pattern Notes

- Strongest active lanes: AI Transformation, Stability & Governance, Business Unit / Integration, Startup / Product Engineering.
- Repeated risk: roles that sound senior but screen for current hands-on coding or narrow stack ownership.
- Repeated mitigation: avoid leading with `200+ engineers` or long-career signals for HH/ATS and smaller teams.
- Business Unit / Integration is now a finalized CV track; older analyses that called it missing are superseded by the current portfolio.
- Effort policy: only `A-class` roles deserve careful tailoring before a human response. `B-class` uses a ready CV and short template. `C-class` stays parked/thin-traced unless explicitly reopened.
