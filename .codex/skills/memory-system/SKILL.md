# Memory System

Use for memory lookup, durable fact capture, semantic summaries, entity notes, retrieval notes, or graph indexing.

## Start

1. Read `wiki/maps/memory-system.md`.
2. Read `memory/README.md`.
3. Read `memory/protocol/lifecycle.md` when the task involves use, enrich, create, change, or retire.
4. If needed, read `wiki/playbooks/memory-system.md`.

## Output Contract

- Keep source-of-truth facts in domain folders first.
- Use `memory/` for summaries, retrieval, entities, graph, and durable context.
- Use `memory/protocol/metadata-contract.md` for new protocol-managed memory.
- Prefer maps, indexes, and narrow concept cards over broad raw-data reads.
- Keep knowledge graph JSONL valid.
